## 二、评测环境硬约束（决定模型选型的核心前提）

| 约束项 | 数值 |
|--------|------|
| CPU 核数 | **2 核** |
| 内存 | **8 GiB** |
| 推理设备 | **CPU 推理**（非 GPU） |
| 总推理时限 | **≤ 70 分钟** |
| 测试集规模 | 约几千张 |

### 倒推结论

按 2000–5000 张测试图估算，**单张推理必须控制在 ~1 秒以内**，模型权重 < 100 MB 才安全。

## 三、技术方案

### 3.1 总体策略

> **ImageNet 预训练 backbone + 全连接分类头微调**

- 利用 ImageNet 预训练权重，模型已具备底层视觉特征提取能力
- 只需在天气数据上做迁移学习（transfer learning），15–20 epoch 即可收敛
- 不从头训练（数据量不足，小类更训不出来）

### 3.2 候选模型（四选一主攻）

| 优先级 | 模型 | 参数量 | 权重体积 | 推荐理由 |
|--------|------|--------|----------|----------|
| 🥇 主攻 | **EfficientNet-B0** | 5.3M | ~20 MB | 精度/速度最优平衡，CPU 推理最快 |
| 🥈 备选 | **ResNet18** | 11.7M | ~45 MB | 最成熟的基线，调试最省心 |
| 🥉 备选 | **MobileNetV3-Large** | 5.4M | ~22 MB | 专为移动端/CPU 设计 |
| 🏅 冲刺 | **ConvNeXt-Tiny** | 28.6M | ~110 MB | 精度可能更高，需重点测 CPU |

### 3.3 模型加载方式

```python
import timm
model = timm.create_model('efficientnet_b0', pretrained=True, num_classes=4)
```

- 首次运行自动下载 ImageNet 预训练权重（缓存到 `~/.cache/torch/hub/`）
- 自动替换分类头为 4 类输出
- 后续运行直接使用缓存，无需联网

### 3.4 数据增强策略

| 阶段 | 操作 | 参数 |
|------|------|------|
| **训练** | RandomResizedCrop | 224×224 |
| | HorizontalFlip | p=0.3 |
| | ColorJitter | brightness=0.1, contrast=0.1, saturation=0.1 |
| | Normalize | ImageNet mean/std |
| **验证/推理** | Resize | 256×256 |
| | CenterCrop | 224×224 |
| | Normalize | ImageNet mean/std |

增强保持**轻量**——天气识别依赖整体场景信息，强旋转、强透视、CutMix/MixUp 不一定有利。

## 四、数据现状与应对

### 4.1 数据集分布

| 类别 | 数量 | 占比 |
|------|------|------|
| cloudy | 2184 | 43.7% |
| rainy | 446 | 8.9% |
| snowy | 403 | 8.1% |
| sunny | 1966 | 39.3% |
| **总计** | **4999** | — |

### 4.2 严重不均衡的应对措施

1. **Loss 层** — `CrossEntropyLoss(weight=class_weights)`，权重按 `1/count` 自动计算，强制模型重视 rainy/snowy
2. **验证层** — Stratified split，保证验证集中各类比例与训练集一致
3. **数据层** — rainy/snowy 可用稍强的 RandAugment 做隐式过采样（备选）

## 八、最容易犯的错误清单

| # | 错误 | 正确做法 |
|---|------|----------|
| 1 | 只看 Accuracy 不看 Macro F1 | 训练中输出 **val_macro_f1**，监控 per-class F1 |

| 5 | 依赖冷门库 | 仅依赖 `torch + torchvision + timm` |
